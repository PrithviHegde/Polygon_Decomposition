Timing analysis

We can see, after experimentation with test-cases of differing edges and sizes, that as our input size increases, we find that the running time of our program also grows. This relationship makes sense, as we can understand that the time taken is exponentially proportional to our input size.

We recorded the running time taken for polygons of varied sizes, from 5 to almost 300 vertices, and plotted it against the input size to get the following graph.
